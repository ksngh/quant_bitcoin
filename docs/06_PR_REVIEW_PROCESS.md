# Purpose

PR review verifies that a task is complete, scoped, tested, safe, and consistent with architecture.

# Review Steps

1. Read requirement file.
2. Read task file.
3. Inspect changed files.
4. Check task scope.
5. Check architecture boundaries.
6. Check tests.
7. Check safety.
8. Check docs/decision updates.
9. Approve, request changes, or create follow-up task.

# Required PR Summary

- Requirement
- Task
- What changed
- What did not change
- Tests run
- Safety notes
- Known limitations
- Follow-up tasks

# Review Decision Types

- Approve
- Request changes
- Follow-up task
