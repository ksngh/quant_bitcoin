# Initial Scope Requirement

# Raw Requirement

The project should start as a small Python Bitcoin quant trading application.
It should first support candle data, RSI strategy, basic backtesting, and paper trading.
It should not implement live trading, real order execution, risk management, dashboard, database, scheduler, machine learning, futures, leverage, or portfolio optimization in the first version.

# Clean Requirement

Set up the project direction and workflow for a small Python Bitcoin quantitative trading application focused first on candle data, RSI strategy, basic backtesting, and paper trading.

# Problem

The project needs a controlled first-version scope before implementation begins.

# User Goal

Future Codex tasks should implement only the first-version trading workflow and avoid unsafe or premature features.

# Functional Requirements

- Define the first-version scope.
- Define out-of-scope areas.
- Require task documents, tests, self-review, and PR review before implementation completion.

# Explicitly Out of Scope

- live trading
- real order execution
- risk management
- dashboard
- database
- scheduler
- machine learning
- futures
- leverage
- portfolio optimization

# Acceptance Criteria

- Workflow docs exist.
- Future task files exist.
- Review checklists exist.
- Safety rules are explicit.
